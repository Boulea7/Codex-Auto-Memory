# Security Policy

## Supported scope

Security fixes are handled on the current mainline of the repository unless a release note says otherwise.

## How to report a vulnerability

Please do not open a public issue with exploit details.

This repository does not currently publish a dedicated security email address in-tree. Use the repository routes below instead of inventing or guessing an email alias.

Preferred route:

1. Use GitHub's private vulnerability reporting flow for this repository if it is available in your view.

Fallback route:

1. Use the contact guidance in [SUPPORT.md](./SUPPORT.md) to request a private reporting path.
2. Do not include secrets, proof-of-concept exploits, or reproduction details in public channels.
3. Share proof-of-concept exploits or reproduction details only through GitHub's private vulnerability reporting flow or another confirmed private reporting route.

## What to include

- affected version, branch, or commit if known
- impact summary
- reproduction steps or proof of concept, but only when using a private reporting route
- any suggested mitigation or patch direction

## Response expectations

- triage is handled on a best-effort basis
- reports may be validated, narrowed, or declined if they are not reproducible
- coordinated disclosure is preferred until a fix or mitigation is ready
