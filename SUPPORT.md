# Support

## Before opening anything

- Start with the landing README in your preferred language: [简体中文](./README.md), [繁體中文](./README.zh-TW.md), [English](./README.en.md), or [日本語](./README.ja.md).
- Use the matching docs hub when you need architecture, integration, host-boundary, or release references.
- Check whether your question is really a bug, a feature request, or a security report.

## Where to go

| Need | Best route |
| :-- | :-- |
| product overview or setup guidance | one of the localized README entry pages plus the corresponding docs hub |
| reproducible defect | use the bug report issue template |
| feature or workflow request | use the feature request issue template |
| vulnerability or security-sensitive concern | follow [SECURITY.md](./SECURITY.md) |
| expected community behavior | read [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md) |

## Private contact fallback

This repository does not currently publish a dedicated support email or security mailbox in-tree.

If GitHub private vulnerability reporting is not available in your view and you need a private route, use any public contact detail published on the repository owner profile:

- <https://github.com/Boulea7>

Do not post sensitive details in public while requesting that route.

## Issue quality

Good issues usually include:

- what you expected
- what happened instead
- exact commands or steps
- environment details that matter
- links to the relevant docs you already checked

## Language

English and Chinese are both acceptable in issues and documentation updates. Keep titles and reproduction steps as explicit as possible.
